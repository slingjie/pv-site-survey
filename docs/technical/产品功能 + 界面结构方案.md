## 一、整体定位与使用流程

**产品定位**：
 面向光伏/储能等项目的现场踏勘人员，用于在现场采集厂区、建筑屋面、电气设施等信息和照片，并最终生成一份「踏勘报告」（大概率是 PDF 或在线报告）。

**核心流程**：

1. 新建／打开一个踏勘报告记录；
2. 在顶部录入项目基础信息（项目编号、所在地区）；
3. 通过顶部 4 个 Tab 依次填写：
   - 厂区概况
   - 建筑屋面
   - 电气设施
   - 收资清单
4. 每个 Tab 里再通过左侧垂直菜单进入对应子模块，拍照、上传资料、选择选项、填写说明；
5. 随时可点击底部「保存」进行草稿保存；
6. 填写完成后，点击「生成踏勘报告」生成正式报告。

## 二、顶层界面与通用布局

### 1. 踏勘报告主界面

**头部区域：**

- 左上：返回图标；
- 中间：标题「踏勘报告」；
- 右上：更多（…）和一个圆形图标（可能是记录列表/相机入口）；
- 标题下：
  - 报告编号（示例：1）；
  - 所在地区选择框（如「浙江省杭州市拱墅区」）。

**模块 Tab（水平）**

- 厂区概况
- 建筑屋面
- 电气设施
- 收资清单

点击任一 Tab，右侧为该模块内容，左侧会出现垂直二级导航。

**底部固定按钮**

- 左：橙色描边按钮「生成踏勘报告」
- 右：黄色实心按钮「保存」

## 三、模块与界面结构（按截图整理去重）

### A. 厂区概况模块

左侧垂直菜单包含 4 个子模块：

1. 位置交通
2. 风险区域
3. 用电信息
4. 产权信息

顶部还有一个大区域用于厂区整体照片（说明文字 + 大号「拍照」卡片）。

------

#### A1. 地理位置（属于「位置交通」里的一个子页）

字段与组件：

- 工厂地址（必填）
  - 文本输入框
  - 右侧「当前位置」按钮（定位填入地址/经纬度/海拔等）。
- 厂区鸟瞰图
  - 上传/拍照组件
  - 右侧有「示例」按钮。
- 房屋位置分布图
  - 上传/拍照组件
  - 右侧「示例」。

> 你可以把「位置交通」下面拆成两个卡片：**地理位置** 和 **交通条件**，都在同一个页面按顺序排布。

------

#### A2. 位置交通（另一部分）

字段与组件：

- 房屋位置分布图
  - 上传/拍照组件（有说明「直接上传有标记的图片…」）。
- 交通条件
  - 单选按钮组：
    - 基本满足
    - 受限
  - 多行文本说明：
    - 说明设备及材料运输的交通情况。

------

#### A3. 风险区域

字段与组件：

- 风险区域标签选择（多选 Chip）：
  - 有毒有害
  - 易燃易爆
  - 高污染
  - 高温
  - 自然灾害
  - - 添加（自定义标签）
- 风险区域说明：多行文本框；
- 风险区域照片：上传/拍照组件。

------

#### A4. 用电信息

字段与组件：

- 用电时段（下拉选择 → 底部弹出列表）
  - 示例选项：
    - 单休 / 日间用电时段多
    - 全天用电均衡
    - 双休 / 夜间用电时段多 ……
- 电价类型（下拉）
  - 例如：工商业、一般工商业、其他。
- 电压等级（下拉）
- 分时电价（单选）
  - 是 / 否
- 电费单（近一年）
  - 点击进入选择收资情况：齐全 / 部分 / 待收 / 无法提供
- 用电负荷曲线（近一年）
  - 同上收资情况选择。
- 用电企业营业执照
  - 同上收资情况选择。

> 这里的三个资料字段，用的是和「收资清单」一致的枚举状态，建议做成一个通用组件。

------

#### A5. 产权信息

字段与组件：

- 土地证
  - 选择收资情况（齐全 / 部分 / 待收 / 无法提供）。
- 产权证 / 查册信息
  - 收资情况选择。
- 业主性质
  - 下拉选择（例如 自有 / 租赁 / 其他）。

------

### B. 建筑屋面模块

左侧垂直菜单：

1. 基础信息
2. 屋顶屋面
3. 内部结构
4. 房屋周边

顶部同样有一块「现场照片」区域，用于房屋整体照片。

此外，建筑屋面模块有「多屋顶」机制：

- 顶部横向标签：
  - 屋顶-1（选中状态）
  - - 新增（添加更多屋顶）

每个屋顶都有自己的一套基础信息、屋面信息、内部结构等数据。

------

#### B1. 基础信息

分为两个部分：**房屋信息** + **其他基础信息**。

1）房屋信息

- 屋顶名称（必填）
- 屋顶面积（必填，单位 m²）
- 屋顶类型（必填，下拉选择 → 底部弹窗）
  - 你给的弹窗截图里大致可以看出几组类型：
    - 彩钢瓦、梯形彩钢、直立锁边彩钢等
    - 水泥屋顶、预制板、现浇钢混、双 T 板 等
  - 建议做成一个可配置枚举字典。

2）其他基础信息（从另一张截图整理）

- 装机容量（必填，MW）
- 房屋鸟瞰图（上传/拍照，带「示例」）
- 朝向
  - 文本 / 下拉 + 「获取」按钮（通过手机罗盘或者地图获取）。

------

#### B2. 屋顶屋面

包括两大块：遮挡物/障碍物 + 屋面污染或锈蚀。

1）遮挡物/障碍物

- 提示说明文字（屋面有无凸出遮挡物…）。
- 屋顶利用率（必填，下拉选择 → 底部弹窗）
  - 示例选项：
    - 障碍物较多
    - 障碍物较少
    - 障碍物无
- 避挡障碍物类型（Chip 多选）
  - 女儿墙
  - 气楼 / 通风口
  - 烟囱 / 锅炉
  - 天窗
  - 采光用
  - 水箱
  - 管道设备
  - - 添加
- 遮挡物照片：上传/拍照。

2）屋面污染或锈蚀

- 污染情况（下拉选择）
  - 示例：污染较重 / 污染一般 / 基本无污染。
- 屋面污染照片：上传/拍照（带「示例」）。

------

#### B3. 内部结构

1）建筑结构

- 说明文字（房屋檩条、钢梁梁柱…）。
- 结构类型（下拉，底部弹窗）；
- 结构照片上传/拍照。

2）异常情况

- 异常类型（Chip 多选）：
  - 裂缝
  - 漏水
  - 下沉
  - 变形
- 异常照片上传/拍照；
- 补充说明（多行文本）：
  - 异常位置、严重程度等。

------

#### B4. 房屋周边

- 周边环境照片：上传/拍照
  - 提示：空地、绿化带、道路等；
- 也可以扩展文本说明字段（如：周边遮挡、退距等）。

------

### C. 电气设施模块

左侧垂直菜单：

1. 现场照片
2. 设施照片
3. 配电信息
4. 并网信息

顶部仍是模块级说明区域。

------

#### C1. 现场照片

- 说明文案（电气设施的现场照片，包括配电房、变压器及铭牌…）。
- 大号「拍照」卡片，用于直接调用相机。

------

#### C2. 设施照片

支持多设施管理：

- 顶部：横向设施标签
  - 1#设施（选中）
  - - 新增（添加 2#、3# 等）
- 当前设施中需上传的图片项（每项一个上传/拍照卡片 +「示例」按钮）：
  - 配电室
  - 变压器及铭牌
  - 一次系统图
  - 配电柜
  - 计量表
  - 周边环境

------

#### C3. 配电信息

字段与组件：

- 电压信息
  - 进线电压（下拉）
  - 计量点电压（下拉）
- 开关站（Chip 单选）：
  - 无开关站
  - 10kV
  - 35kV
- 变压器
  - 变压器信息：+添加（点击后进入「变压器明细」页面，未在截图中展示，可自行设计：容量、台数、型号、接线方式等）
  - 变压器位置分布图：上传/拍照（说明需在厂区鸟瞰图上标注）。

------

#### C4. 并网信息

拆分为两块内容：

1）并网基本信息

- 上网模式（必填，下拉）；
- 并网柜位置（多行文本：拟定并网柜安装位置）；
- 新增电缆敷设方式和路径（多选勾选框）：
  - 电缆沟
  - 桥架
  - 直埋
  - 顶管
  - 其他
- 补充说明（多行文本）：电缆走向和路径长度；
- 相关照片上传（底部有上传/拍照区域）。

2）接入方案与风险

- 拟定接入（并网等级）（下拉选择）；
- 接入方案描述（多行文本）；
- 注意事项：
  - 多行文本，内容包含环境影响、施工成本、安全风险等。

------

### D. 收资清单模块

这个模块是统一管理「资料是否已收齐」的矩阵界面。

结构：

- 表头：
  - 资料名称
  - 齐全
  - 部分
  - 待收
  - 无法提供
- 下方按类别分组：

1）厂区 / 厂房资料

- 厂区总平图
- 厂区管线分布图
- 厂区地勘资料
- 建筑设计总说明
- 建筑施工图整册
- 结构施工图整册
- 屋顶暖通管线布置图
- 屋顶消防管线布置图

2）资质 / 产权

- 用电企业营业执照
- 土地证
- 房产证

3）用电信息（近一年）

- 电费单
- 用电负荷曲线

4）电气资料

- 电气总平面图
- 电气设计总说明
- 高低压一次系统图
- 防雷接地图
- 电缆走线图
- 配电室平面布置图

每行都是单选（同一资料在 4 个状态中选 1 个），可通用到其他模块中（如前面「用电信息」和「产权信息」里的字段）。

## 四、通用交互与组件规范

1. **必填标记**：字段名前用「*」标出，如 *屋顶名称、*屋顶面积、*装机容量、*工厂地址 等。
2. **下拉选择**：
   - 采用「底部弹出选择框」样式；
   - 下方两个按钮：取消（灰底）、确定（绿色）。
3. **单选 / 多选**：
   - 单选：圆形 Radio（例如 分时电价 是/否、交通条件）；
   - 多选：圆角 Chip（例如 风险区域类型、遮挡物类型、异常情况）。
4. **图片上传**：
   - 统一使用虚线边框 + 中间「+ 上传/拍照」；
   - 某些关键图片右上角有「示例」链接，点击打开样例图片。
5. **多实例管理**：
   - 屋顶：顶部「屋顶-1」+「+新增」；
   - 电气设施：顶部「1#设施」+「+新增」；
   - 变压器：列表 +「+添加」按钮。
6. **保存与生成报告**：
   - 任意页面均可点击底部固定按钮；
   - 「保存」只保存当前报告草稿；
   - 「生成踏勘报告」进入报告生成流程（预览/导出）。

## 五、复刻实现建议（概要）

如果你后面要落地开发，我建议：

1. **信息模型设计（简化版）**

- Report（踏勘报告）
  - 基础信息：编号、地址、省市区等；
  - PlantOverview（厂区概况子对象）；
  - Roofs[]（多个屋顶，每个屋顶包含基础信息、屋面情况、内部结构、周边环境）；
  - Facilities[]（多个电气设施，包括照片、配电信息、并网信息等）；
  - CollectionStatus（收资清单整体结构或按资料项存状态）。

1. **界面结构**

- 报告编辑页：对应你截图的这个「踏勘报告」主容器；
- 其它都是在这个编辑页内部切 Tab 和二级菜单，尽量不要跳太多新页面（除了比如「变压器明细」这样的弹出页）。

1. **组件复用**

重点把以下做成可复用组件：

- 地址选择 + 当前定位；
- 图片上传卡片（带示例）；
- Chip 多选组件；
- 底部弹窗选择器（支持单选列表、分组列表）；
- 收资状态组件（齐全/部分/待收/无法提供）。
